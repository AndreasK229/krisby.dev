About

I’m Andreas Krisby, based in Malmö, currently working in healthcare IT while trying to move into IAM and cloud security.

A lot of my interest in identity and access management actually comes from working around sensitive systems and patient data. In healthcare, access control is a real-world problem — who should have access, when they should have it, and how to make sure the wrong people don’t. That made me interested in how identity, permissions, authentication, and governance work behind the scenes.

Most of what I learn comes from building labs, testing things myself, breaking configurations, and figuring out why something works — or doesn’t. This site is mainly where I document that process.

Right now I spend most of my time learning more about Microsoft Entra ID, Conditional Access, hybrid identity, authentication flows, identity governance, and cloud security fundamentals.

Things I keep coming back to:

  • authentication and federation
  • access control
  • provisioning
  • Zero Trust
  • hybrid identity
  • cloud governance

I’m less interested in “perfect” demo environments and more interested in understanding how identity systems actually behave in practice.

How I Learn

I learn best by building things myself.

Instead of only reading documentation or watching courses, I usually create environments, test scenarios, troubleshoot problems, and write down what I learn along the way.

That process has helped me understand identity and access management much better — especially how authentication, permissions, provisioning, and security controls connect together in real environments.

Current Goal

Right now my goal is to keep improving technically and eventually move fully into IAM and cloud security work, preferably in a role where I can keep learning while working hands-on with identity systems and modern cloud environments.
