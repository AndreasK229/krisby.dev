AI RBAC Analyzer

By Andreas Krisby 2026-03-13 2 min read

I wanted to understand Azure RBAC better in practice and at the same time test if AI can be used as support to find over-privileged access and risks faster. The goal was not to build a perfect tool, but to combine theory with practical testing.

Step 1 – Idea and purpose

Analyze Azure RBAC assignments with a local LLM. The model should:

  • Identify broad access
  • Analyze scope
  • Give improvement suggestions based on least privilege

Step 2 – Create test environment in Azure

Created:

  • A resource group
  • Virtual machines
  • Network resources

Created users and groups to simulate a real environment:

  • vm-tech
  • vnet-tech

Step 3 – Assign RBAC roles

Gave different roles to the groups at resource group level:

  • vm-tech → Virtual Machine Contributor
  • vnet-tech → Network Contributor

Step 4 – Export RBAC data

Created example-rbac.json with RBAC information:

[
  {
    "principalName": "vm-tech",
    "roleDefinitionName": "Virtual Machine Contributor",
    "scope": "/subscriptions/.../resourceGroups/RG-TEST-LAB"
  },
  {
    "principalName": "vnet-tech",
    "roleDefinitionName": "Network Contributor",
    "scope": "/subscriptions/.../resourceGroups/RG-TEST-LAB"
  }
]

Step 5 – Write the prompt

Created prompt.txt where the model was instructed to:

  • Analyze roles
  • Identify broad access
  • Analyze scope
  • Give improvement suggestions

Step 6 – Install Ollama

ollama pull llama3

Step 7 – Build the PowerShell script

Created run-agent.ps1:

$prompt = Get-Content -Raw "prompt.txt"
$data   = Get-Content -Raw "example-rbac.json"
$input  = "$prompt`n`nRBAC DATA:`n$data"

$input | ollama run llama3

The script:

  1. Reads prompt.txt
  2. Reads example-rbac.json
  3. Sends everything combined to Ollama

Step 8 – Run the analysis

.\run-agent.ps1

The AI then analyzed roles, scope, possible risks and gave least privilege suggestions.

Step 9 – Lessons

The project clarified:

  • How broad the Contributor role is
  • How important scope is — resource group vs. subscription makes a big difference
  • How easy it is to give too much access
  • How important least privilege is in practice

AI can be good support for getting a quick overview, but you need to understand what the results mean yourself.

Step 10 – Possible next steps

  • Fetch RBAC directly from Azure with Get-AzRoleAssignment
  • Analyze larger environments
  • Improve prompts for more accurate recommendations
