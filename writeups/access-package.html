Microsoft Entra ID – Access Package Lab

Syfte

Detta labb demonstrerar hur Identity Governance och Entitlement Management används i Microsoft Entra ID för att:

  • Ge extern access på ett kontrollerat sätt
  • Automatisera onboarding av gästanvändare
  • Minska manuell administration
  • Implementera least privilege
  • Få bättre spårbarhet och governance

Vad detta löser för problem

  • Manuell hantering av externa användare
  • Överbehörigheter
  • Ingen kontroll över vem som har access
  • Svårt att följa upp gamla guest-konton
  • Ingen expiration på access
  • Ingen audit trail över access requests
  • Ingen approval-process

Access Packages gör det möjligt att automatisera och standardisera access till:

  • Microsoft Teams
  • SharePoint
  • Security Groups
  • Applikationer
  • Azure-resurser

Risker om detta INTE används

  • Gamla externa konton ligger kvar med access
  • Överprivilegierade användare
  • Shadow IT
  • Ingen lifecycle management
  • Svårt att följa compliance-krav
  • Ingen central governance
  • Risk för data exposure
  • Ingen kontroll över guest access

I större miljöer leder detta ofta till:

  • Privilege creep
  • Orphaned accounts
  • Ökad attackyta
  • Compliance-problem

Steg 1 – Skapa Security Group

  1. Logga in i Microsoft Entra Admin Center
  2. Gå till Identity → Groups
  3. Klicka New Group
  4. Group type: Security
  5. Namn: Medical-Readers
  6. Membership type: Assigned eller Dynamic User

Exempel på dynamisk regel

(user.userType -eq "Guest")

Steg 2 – Skapa katalog

  1. Gå till Identity Governance
  2. Välj Catalogs
  3. Klicka New catalog
  4. Namn: Medical Reader Catalog

Steg 3 – Lägg till resurser

  1. Öppna katalogen
  2. Välj Resources
  3. Klicka Add resources
  4. Välj Security Group
  5. Lägg till gruppen

Steg 4 – Skapa Access Package

  1. Gå till Access Packages
  2. Klicka New access package
  3. Namn: Medical Readers
  4. Välj katalog
  5. Lägg till gruppen som resource role

Steg 5 – Skapa Assignment Policy

  1. Öppna Access Package
  2. Välj Policies
  3. Klicka New policy

Inställningar

  • Users who can request access: For users not in your directory
  • Approval: No approval required
  • Expiration: 1 day
  • Enable requests: Enabled

Steg 6 – Skicka invite

  1. Öppna Access Package
  2. Kopiera Request link
  3. Skicka länken till extern användare

När användaren accepterar:

  • Guest-konto skapas
  • Användaren får gruppmedlemskap
  • Access provisioneras automatiskt

Steg 7 – Verifiera provisioning

Kontrollera:

  • Entra ID → Users → Guest Users
  • Groups → Members
  • Access Package → Assignments
  • Audit Logs
  • Sign-in Logs
