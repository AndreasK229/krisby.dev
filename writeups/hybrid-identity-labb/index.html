Hybrid identity Lab

By Andreas Krisby 2026-01-20 2 min read

Build a hybrid identity between on-prem AD and Entra and test how users, groups and sync work in practice.

Environment: Windows Server 2022 · Oracle VirtualBox · Active Directory · Microsoft Entra ID · Entra Connect

Step 1 – Install Windows Server 2022

  1. Create a new VM in Oracle VirtualBox.
  2. Install Windows Server 2022.
  3. Give the server a static IP.
  4. Rename the server.
  5. Restart the server.

Step 2 – Install Active Directory

  1. Open Server Manager.
  2. Click Add Roles and Features.
  3. Install Active Directory Domain Services.
  4. Promote the server to Domain Controller.
  5. Create domain: lab.local
  6. Restart the server.

Step 3 – Create OU structure

In Active Directory Users and Computers, create the following OUs:

  • HR
  • IT
  • Groups

OUs are used for structure. Groups are used for access.

Step 4 – Create users

HR: Anna HR, Erik HR
IT: Johan IT, Sara IT

Place HR users in OU=HR and IT users in OU=IT.

Step 5 – Create global security groups

In OU=Groups, create groups with Group scope = Global and Group type = Security:

  • GG_HR_Read
  • GG_IT_Admin
  • GG_App_Reader

Step 6 – Add users to groups

Anna HR  → GG_HR_Read
Johan IT → GG_IT_Admin

Principle: User → Group → Access

Step 7 – Install Entra Connect

  1. Download Microsoft Entra Connect.
  2. Install the software.
  3. Sign in with Entra Global Admin.
  4. Connect the local AD domain.
  5. Enable Password Hash Sync.
  6. Complete the installation.

Step 8 – Test sync

Open Microsoft Entra Admin Center and verify that users, groups and group memberships are visible.

Step 9 – Run manual sync

Import-Module ADSync

# Delta sync
Start-ADSyncSyncCycle -PolicyType Delta

# Initial sync
Start-ADSyncSyncCycle -PolicyType Initial

Step 10 – Troubleshooting

Problem: Sync failed.
Error: Certificate/token authentication didn’t work.
Cause: Wrong system time (NTP).
Lesson: Auth flow is sensitive to correct time.

Step 11 – Test changes

Tested:

  • Adding users to groups
  • Removing users from groups
  • Changing users
  • Disabling users

After each change, Delta sync was run and the change was verified in Entra.

Step 12 – RBAC thinking

RBAC should go through groups, not directly on individual users:

User → Group → Access
GG_App_Reader → Reader access

Lessons

  • AD is source of truth — Entra mirrors identities from on-prem
  • Entra Connect is controlled by sync rules
  • RBAC should go through groups
  • Delta vs Initial sync affects how quickly changes take effect
  • Auth/token flows depend on correct system time
