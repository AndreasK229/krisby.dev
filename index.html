Writeup — Entra ID + Cloudflare Access + SCIM

Satte upp Entra ID som IdP mot Cloudflare Access för min domän för att förstå hur auth, provisioning och Zero Trust faktiskt fungerar i praktiken.

Detta används

  • Microsoft Entra ID
  • Cloudflare Zero Trust
  • OIDC federation
  • SCIM provisioning

Flöde

Entra ID → OIDC → Cloudflare Access
Entra ID → SCIM → Cloudflare

Vad setupen gör

  • Entra ID fungerar som IdP
  • Users och grupper syncas automatiskt via SCIM
  • Access kan styras gruppbaserat
  • Deprovisioning sker automatiskt när användare tas bort

Problem under setup

  • Redirect URI mismatch
  • Fel Application ID
  • Invalid client secret
  • API permissions och admin consent

Lärdomar

  • SCIM förenklar provisioning och deprovisioning rejält
  • Gruppbaserad access sparar mycket manuellt arbete
  • Det är mycket lättare att förstå identity-flöden när man ser systemen prata med varandra live
