AI RBAC Analyzer

By Andreas Krisby 2026-03-13 2 min read

Jag ville förstå Azure RBAC bättre i praktiken och samtidigt testa om AI kan användas som stöd för att snabbare hitta överbehörighet och risker i access. Målet var inte att bygga ett perfekt verktyg, utan att kombinera teori med praktisk testning.

Steg 1 – Idé och syfte

Analysera Azure RBAC-tilldelningar med en lokal LLM. Modellen ska:

  • Identifiera bred access
  • Analysera scope
  • Ge förbättringsförslag utifrån least privilege

Steg 2 – Skapa testmiljö i Azure

Skapade:

  • En resource group
  • Virtuella maskiner
  • Nätverksresurser

Skapade users och grupper för att simulera en verklig miljö:

  • vm-tech
  • vnet-tech

Steg 3 – Tilldela RBAC-roller

Gav olika roller till grupperna på resource group-nivå:

  • vm-tech → Virtual Machine Contributor
  • vnet-tech → Network Contributor

Steg 4 – Exportera RBAC-data

Skapade example-rbac.json med RBAC-information:

[
  {
    "principalName": "vm-tech",
    "roleDefinitionName": "Virtual Machine Contributor",
    "scope": "/subscriptions/.../resourceGroups/RG-TEST-LAB"
  },
  {
    "principalName": "vnet-tech",
    "roleDefinitionName": "Network Contributor",
    "scope": "/subscriptions/.../resourceGroups/RG-TEST-LAB"
  }
]

Steg 5 – Skriva prompten

Skapade prompt.txt där modellen instruerades att:

  • Analysera roller
  • Identifiera bred access
  • Analysera scope
  • Ge förbättringsförslag

Steg 6 – Installera Ollama

ollama pull llama3

Steg 7 – Bygga PowerShell-scriptet

Skapade run-agent.ps1:

$prompt = Get-Content -Raw "prompt.txt"
$data   = Get-Content -Raw "example-rbac.json"
$input  = "$prompt`n`nRBAC DATA:`n$data"

$input | ollama run llama3

Scriptet:

  1. Läser prompt.txt
  2. Läser example-rbac.json
  3. Skickar allt samlat till Ollama

Steg 8 – Köra analysen

.\run-agent.ps1

AI:n analyserade sedan roller, scope, möjliga risker och gav least privilege-förslag.

Steg 9 – Lärdomar

Projektet tydliggjorde:

  • Hur bred Contributor-rollen är
  • Hur viktigt scope är — resource group vs. subscription gör stor skillnad
  • Hur lätt det är att ge för mycket access
  • Hur viktigt least privilege är i praktiken

AI kan vara ett bra stöd för att snabbt få en överblick, men man måste själv förstå vad resultatet betyder.

Steg 10 – Möjliga nästa steg

  • Hämta RBAC direkt från Azure med Get-AzRoleAssignment
  • Analysera större miljöer
  • Förbättra prompts för mer precisa rekommendationer
