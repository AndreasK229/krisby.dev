Hybrid identity Lab

By Andreas Krisby 2026-01-20 2 min read

Bygga en hybrid identity mellan on-prem AD och Entra och testa hur users, grupper och sync fungerar i praktiken.

Miljö: Windows Server 2022 · Oracle VirtualBox · Active Directory · Microsoft Entra ID · Entra Connect

Steg 1 – Installera Windows Server 2022

  1. Skapa ny VM i Oracle VirtualBox.
  2. Installera Windows Server 2022.
  3. Ge servern statisk IP.
  4. Byt namn på servern.
  5. Starta om servern.

Steg 2 – Installera Active Directory

  1. Öppna Server Manager.
  2. Klicka Add Roles and Features.
  3. Installera Active Directory Domain Services.
  4. Promota servern till Domain Controller.
  5. Skapa domän: lab.local
  6. Starta om servern.

Steg 3 – Skapa OU-struktur

I Active Directory Users and Computers, skapa följande OU:

  • HR
  • IT
  • Groups

OU används för struktur. Grupper används för access.

Steg 4 – Skapa users

HR: Anna HR, Erik HR
IT: Johan IT, Sara IT

Placera HR-users i OU=HR och IT-users i OU=IT.

Steg 5 – Skapa globala säkerhetsgrupper

I OU=Groups, skapa grupper med Group scope = Global och Group type = Security:

  • GG_HR_Read
  • GG_IT_Admin
  • GG_App_Reader

Steg 6 – Lägg users i grupper

Anna HR  → GG_HR_Read
Johan IT → GG_IT_Admin

Principen: User → Group → Access

Steg 7 – Installera Entra Connect

  1. Ladda ner Microsoft Entra Connect.
  2. Installera programmet.
  3. Logga in med Entra Global Admin.
  4. Koppla lokal AD-domän.
  5. Aktivera Password Hash Sync.
  6. Slutför installationen.

Steg 8 – Testa sync

Öppna Microsoft Entra Admin Center och verifiera att users, grupper och gruppmedlemskap syns.

Steg 9 – Kör manuell sync

Import-Module ADSync

# Delta sync
Start-ADSyncSyncCycle -PolicyType Delta

# Initial sync
Start-ADSyncSyncCycle -PolicyType Initial

Steg 10 – Felsökning

Problem: Sync failade.
Fel: Certifikat/token-autentisering fungerade inte.
Orsak: Fel systemtid (NTP).
Lärdom: Auth-flödet är känsligt för korrekt tid.

Steg 11 – Testa förändringar

Testade:

  • Lägga users i grupper
  • Ta bort users ur grupper
  • Ändra users
  • Disable users

Efter varje ändring kördes Delta sync och förändringen verifierades i Entra.

Steg 12 – RBAC-tänk

RBAC bör gå via grupper, inte direkt på enskilda users:

User → Group → Access
GG_App_Reader → Reader-access

Lärdomar

  • AD är source of truth — Entra speglar identiteter från on-prem
  • Entra Connect styrs av sync rules
  • RBAC bör gå via grupper
  • Delta vs Initial sync påverkar hur snabbt förändringar slår igenom
  • Auth/token-flöden är beroende av korrekt systemtid
