JML Monitor Lab

By Andreas Krisby 2026-02-10 3 min read

Byggde en enkel JML-monitor mot Microsoft Entra ID för att förstå Joiner/Mover/Leaver, gruppbaserad RBAC, hur access förändras över tid, och hur man kan upptäcka access drift. Och för att förstå hur Graph API fungerar i praktiken.

Steg 1 – Skapa Entra-labb

Skapade users i Microsoft Entra ID:

  • HR users
  • IT users
  • Admin users
  • Production users

Skapade grupper: HR, Admin, Production, Technical administrator VM, VNET TECH.

Lade users i olika grupper för att simulera RBAC.

Steg 2 – Registrera app i Entra

  1. Entra ID → App registrations → New registration
  2. Ge appen ett namn, t.ex. JML-Monitor
  3. Spara Tenant ID och Client ID efter skapandet

Steg 3 – Skapa client secret

  1. Certificates & secrets → New client secret
  2. Spara secret-värdet direkt (visas bara en gång)

Steg 4 – API Permissions

Lägg till Microsoft Graph-permissions:

  • User.Read.All
  • Group.Read.All
  • Directory.Read.All
  • AuditLog.Read.All

Ge admin consent.

Steg 5 – Installera Python

pip install requests

Steg 6 – Skapa Python-script

Skapa filen get_token.py som hanterar autentisering och API-anrop.

Notering: Kodskelettet för Python-scriptet skrevs med AI-hjälp (Claude). Logiken och arkitekturen — hur Graph API integreras, vilka permissions som krävs, hur man detekterar access drift — är baserad på min förståelse för identity systems och Entra ID. AI-hjälpen var ett verktyg för att konvertera logiken till fungerande kod.

Steg 7 – Auth mot Entra

Autentisering mot Microsoft Identity Platform med client credentials:

url = f"https://login.microsoftonline.com/{tenant_id}/oauth2/v2.0/token"

data = {
    "grant_type": "client_credentials",
    "client_id": client_id,
    "client_secret": client_secret,
    "scope": "https://graph.microsoft.com/.default"
}

response = requests.post(url, data=data)
token = response.json()["access_token"]

Steg 8 – Hämta users

response = requests.get(
    "https://graph.microsoft.com/v1.0/users",
    headers={"Authorization": f"Bearer {token}"}
)
users = response.json()

Fick tillbaka: userPrincipalName, id, displayName.

Steg 9 – Loopa users

for user in users["value"]:
    name = user["displayName"]
    upn  = user["userPrincipalName"]
    uid  = user["id"]

Steg 10 – Hämta grupper per user

url = f"https://graph.microsoft.com/v1.0/users/{user_id}/memberOf"
groups = requests.get(url, headers=headers).json()

Steg 11 – Loopa grupper

user_groups = []
for g in groups.get("value", []):
    user_groups.append(g["displayName"])

Steg 12 – Skapa detektionslogik

If-logik för att flagga risk:

  • Admin access
  • HR + Admin (SoD-konflikt)
  • Disabled konto med aktiv access
  • Nya grupper sedan senaste scan
  • Borttagen access
  • Access som låg kvar efter offboarding

Steg 13 – Skapa snapshot

Spara aktuellt state till state.json:

{
  "user-id-123": {
    "name": "Anna HR",
    "groups": ["HR", "GG_HR_Read"]
  }
}

Steg 14 – Jämföra state

Vid ny körning:

  1. Läs gammal state.json
  2. Hämta ny data från Graph
  3. Jämför: nya grupper, borttagna grupper, förändringar

Steg 15 – Exempel på detektion

Scriptet kunde visa:

⚠️  NY ACCESS       Anna HR      → Admin
🚨  BORTTAGEN ACCESS Johan IT     ← Production
🚨  RISK: Admin access           Sara IT
🚨  RISK: SoD-konflikt           Erik HR (HR + Admin)

Steg 16 – Skydda secret

Insåg att secret hade varit exponerat. Åtgärder:

  • Nytt secret
  • Flytta secret till .env
  • Lägg till .env i .gitignore

Steg 17 – Resultat

Monitorn kunde nu:

  • Övervaka gruppförändringar
  • Upptäcka access drift
  • Visa Mover/Leaver-scenarion
  • Analysera RBAC via grupper
  • Jämföra historik över tid

Lärdomar

  • RBAC blir tydligare via grupper
  • Historik behövs för att upptäcka förändringar
  • Access drift syns först vid jämförelse
  • JML blir tydligt i praktiken
  • Graph API ger mycket värdefull data
  • Security handlar mycket om logik och struktur

Vidareutveckling

  • Teams alerts
  • Audit log monitor
  • Sign-in detection
  • Risk scoring
  • Identity monitoring agent
  • Conditional Access integration
